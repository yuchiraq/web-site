# Веб-сайт для строительной компании "АВАЮССТРОЙ"

Этот проект представляет собой официальный веб-сайт для частного строительного унитарного предприятия "АВАЮССТРОЙ", работающего в Бресте и Брестской области.

## Описание

Веб-сайт служит визитной карточкой компании, предоставляя подробную информацию о ее деятельности, услугах и преимуществах. Основная цель сайта — привлечение новых клиентов и информирование существующих о возможностях компании.

### Основные разделы и функции:

*   **Главная страница:** Содержит общую информацию о компании, ее ключевых преимуществах и перечень основных услуг.
*   **О компании:** Подробное описание истории компании, ее миссии, ценностей и команды.
*   **Услуги:** Детальное описание каждого вида предоставляемых услуг.
*   **Наши проекты:** Галерея с фотографиями выполненных работ, демонстрирующая опыт и качество.
*   **Контакты:** Контактная информация, включая адрес, телефон и форму обратной связи.

## Технологии

*   **Бэкенд:** Go (с фреймворком Gin)
*   **Фронтенд:** HTML, CSS, JavaScript

## Запуск проекта

1.  Установите зависимости Go:
    ```bash
    go mod tidy
    ```
2.  Запустите сервер:
    ```bash
    go run .
    ```
3.  Откройте в браузере `http://localhost:8088`.

## Как это работает

### Архитектура

Сайт представляет собой **монолитный веб-сервер**, написанный на языке **Go** с использованием фреймворка **Gin**. "Монолитный" означает, что вся логика (отображение страниц, обработка форм, SEO-генерация) находится в одном приложении.

Ключевая особенность проекта — он **не использует базу данных** для хранения контента. Вместо этого, он работает по принципу "один HTML-файл — одна страница". Весь контент хранится непосредственно в `*.html` файлах в папке `templates`. Это делает сайт очень быстрым и простым в обслуживании.

### Жизненный цикл запроса

1.  **Запуск**: При старте (`go run .`) приложение инициализирует веб-сервер Gin.
2.  **Автоматическая регистрация маршрутов**: Функция `registerRoutes` сканирует папку `templates`, находит все `.html` файлы и автоматически создает для них соответствующие URL-адреса (маршруты). Например, для `templates/services/drainage.html` создается маршрут `/services/drainage`.
3.  **Загрузка шаблонов**: Все HTML-файлы загружаются в память как именованные шаблоны для последующего рендеринга.
4.  **Обработка запроса**: Когда пользователь заходит, например, на `/services/drainage`, Gin находит соответствующий обработчик.
5.  **Рендеринг HTML**: Обработчик вызывает рендеринг шаблона `drainage.html`. Движок шаблонов Go "собирает" финальную страницу, подставляя в основной шаблон переиспользуемые части (`{{template "head"}}`, `{{template "header"}}` и т.д.) и данные (например, SEO-заголовок).
6.  **Ответ**: Готовая HTML-страница отправляется в браузер пользователя.

### Описание Go-файлов

*   `main.go`: **Точка входа и "дирижер".** Инициализирует все приложение, настраивает логирование, определяет маршруты для статических файлов (CSS, JS, изображения) и запускает веб-сервер.
*   `registerRoutes.go`: **Автоматический создатель страниц.** Сканирует папку `templates` и создает из HTML-файлов маршруты для Gin. Управляет SEO-заголовками и загружает HTML-шаблоны в память.
*   `handlers.go`: **Обработчик форм и уведомлений.** Принимает данные с контактной формы (`/submit`), проверяет их и отправляет уведомления о новых заявках в Telegram (и опционально на email) используя учетные данные из `settings.json`.
*   `photo_handler.go`: **Обработчик главной страницы.** Отвечает за логику главной страницы (`/`). Он считывает фотографии из `static/images/photos`, перемешивает их в случайном порядке для карусели и передает в шаблон `index.html`.
*   `sitemap.go`: **SEO-помощник (Карта сайта).** При запуске сервера генерирует файл `static/data/sitemap.xml`. Этот файл помогает поисковым системам (Google, Yandex) лучше понимать структуру сайта и эффективнее его индексировать.
*   `generateRSS.go`: **SEO-помощник (Yandex Turbo/RSS).** Генерирует специальный RSS-фид для сервиса "Турбо-страницы" Яндекса. Он "заходит" в HTML-файлы страниц услуг, извлекает из них контент (заголовки, текст, картинки) и формирует из него специальный XML-файл. Это позволяет Яндексу показывать пользователям быстрые, адаптированные версии страниц прямо в результатах поиска.
*   `go.mod` и `go.sum`: Файлы управления зависимостями проекта в Go.
